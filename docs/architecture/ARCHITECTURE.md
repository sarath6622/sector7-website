## SEO Architecture

> Added: Phase 10 (2026-03-09)

### Metadata strategy

Every page exports a `metadata` object. Static pages use the `generateMetadata()` helper from `src/lib/seo.ts` which automatically populates:
- `<title>` (with `%s | Sector 7 Gym Kochi` template from root layout)
- `<meta name="description">`
- `<link rel="canonical">`
- `og:title`, `og:description`, `og:image`, `og:url`, `og:type`
- `twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`

Dynamic pages (`/trainers/[slug]`, `/blog/[slug]`) use `async generateMetadata()` to fetch real data from Sanity before generating tags.

### JSON-LD structured data

JSON-LD is rendered as `<script type="application/ld+json">` directly in page JSX вЂ” never in `<head>`.

| Location | Schema | Key properties |
|---|---|---|
| Root layout (`layout.tsx`) | `LocalBusiness` | name, address, geo, telephone, openingHours |
| Homepage (`page.tsx`) | `WebSite` + `SearchAction` | Enables sitelinks search box signal |
| `/trainers/[slug]` | `Person` | name, jobTitle, worksFor, image, hasCredential |
| `/blog/[slug]` | `Article` | headline, author, publisher, image, datePublished |
| `/pricing` | `FAQPage` | All FAQ question/answer pairs |
| `/pricing` | `Product` + `Offer` | One per membership plan, with price in INR |

All JSON-LD is generated via `generateJSONLD()` from `src/lib/seo.ts`.

### Sitemap

Generated automatically by `next-sitemap` as a **postbuild** script (`npm run build` в†’ `next-sitemap`).

Output: `public/sitemap.xml` (index) + `public/sitemap-0.xml` (entries)

Config: `next-sitemap.config.js` вЂ” excludes `/api/*` and `/studio/*`, sets per-page priority and changefreq.

After deploying, submit `https://sector7gym.com/sitemap.xml` to Google Search Console.

### Analytics scripts

Both scripts are loaded via `next/script` with `strategy="afterInteractive"` вЂ” they never block rendering.

- **GA4** (`GoogleAnalytics` component): loads if `NEXT_PUBLIC_GA_MEASUREMENT_ID` is set. Silent no-op otherwise.
- **Microsoft Clarity** (`MicrosoftClarity` component): loads if `NEXT_PUBLIC_CLARITY_PROJECT_ID` is set. Silent no-op otherwise.
- Both are rendered at the bottom of `<body>` in `layout.tsx`.
- GA4 events must go through `trackEvent()` in `src/lib/analytics.ts` вЂ” never call `gtag()` directly.

## Security Architecture

- HTTPS enforced (Vercel default)
- CSP headers in `next.config.ts`
- Rate limiting on `/api/contact`, `/api/trial`, `/api/newsletter` (3 req/IP/hour)
- Honeypot fields on all public forms
- Server secrets never prefixed with `NEXT_PUBLIC_`
- All user inputs sanitized server-side before email dispatch
